Soot笔记
event Posted on: 2020-11-20 event_available Edited on: 2021-01-20 folder In: Analysis visibility Views: 

soot

soot是麦吉尔大学Sable研究小组的一个产品，该小组的目标是提供工具，帮助人们更好地理解和更快地执行Java程序。

数据结构

  • Scene类表示分析发生的完整环境。通过它可以设置应用程序类(提供给Soot用于分析的类)、main类(包含main方法的类)、提取过程间分析的信息(例如，指向信息和调用图)。

  • SootClass类表示加载到Soot或使用Soot创建的单个类。

  • SootMethod表示类的单个方法

  • SootField表示类的成员字段

  • Body表示一个方法体（Method Body），有不同的风格，对应于不同的IR(例如，JimpleBody）

这些数据结构是使用面向对象的技术实现的，并且设计得易于使用和尽可能通用。

soot基本数据结构

在本节中，我们描述了在Soot使用过程中通常要用的基本对象。

Method bodies

soot的Body类对应的是Java的一个函数体，Graph、Flow的构建都是基于Body进行分析。soot提供了4中不同的中间语言，分别用不同的body表示，分别为：BafBody,JimpleBody,ShimpleBody, GrimpBody。soot主要是基于Jimple进行分析，在流程中构建的是JimpleBody,其它的Body的构建需要通过开关来控制。

我们可以使用Body类来访问各种信息，比如：

  1. getLocals() 检索局部变量的声明

  2. getUnits() 检索组成Body的语句

  3. getTraps() 检索Body中所有异常处理句柄

Statement

Soot中的Unit接口表示语句（Statement），不同的IR有不同的实现（例如Jimple使用Stmt，而Grimp使用Inst）

  • getUseBoxes() 检索value

  • getDefBoxes() 检索define

  • getUseAndDefBoxes() 同时检索define和value

  • getBoxesPointingToThis() 检索指向该statement的statement

  • getUnitBoxes() 检索该statement指向的Statement

Unit同样也提供了各种方法查找分支状况，例如fallsThrough和branches。

Value

soot中Values代表一个变量，

  • Local 局部变量
  • Jimple.Constant 常量
  • Jimple.Expr 表达式
  • …………

一个表达式会有多种实现，比如BinopExpr和InvokeExpr，但总来说他们都可以被视为对一个或多个值执行某个操作并返回一个值。

References

soot中的box代表引用，提供了一个间接访问soot (Unit,Value)的入口，类似于Java的一个引用，当Unit包含另一个Unit的时候，需要通过Box来访问，Soot 里提供了两种类型的Box, 一个是ValueBox一个是UnitBox

  • UnitBoxes：refer to Units，分支中一个Unit有多个后继时使用。一个UnixBox包含着一个Unit操作，一个Unit的操作会包含着多个UnitBox（有些类型的Unit会包含其他Unit的引用,那么这时候就需要包含Unixbox），比如说，GotoStmt 需要知道目标的Unit是什么，所以一个Unit会包含其它的UnitBox，通过 UnitBox获取下一个Unit

  • ValueBoxes：与Unit类似，我们通常也需要指向Value的box。这被表示为ValueBox类。对于一个Unit,我们能够得到一系列的ValueBox ,包含了在Unit中使用的值以及定义的值, 一个Valuebox对应这一个Value，而一个value可以对应多个ValueBox

下图的i0<0的表达式就是包含一个valuebox, conditionexprbox，里面包含着LtExpr的value，而LtExpr又包含这op1Box左边和op2Box右边两个ValueBox, op1Box的valuebox又包含着i0这个local的本地value, 以及op2Box的ValueBox又包含着0这个常量Value

Intermediate Representations (IR)

soot提供了4种不同的IR（中间表示），每种IR都有不同的抽象级别，它们分别是：Baf，Grimp，Jimple，Shimple。

Baf:一种简化的字节码表示，操作起来很简单。

Jimple:一种有类型的、3地址的、基于语句的中间表示

Shimple: Jimple的SSA变体。

Grimp:适合反编译和代码检查的Jimple聚合版本。

其中Jimple是Soot的核心，是四种IR中最重要的。Soot能直接创建Jimple码，也可由Java sourcecode或者bytecode转化翻译而来。这里主要介绍Jimple的使用方法。

可以在Soot中直接创建Jimple表示，也可以基于Java源代码和java字节码或类文件。

java字节码会先转换为无类型的Jimple，通过为隐式堆栈位置引入新的局部变量，并使用Subroutine elimination来删除jsr指令。无类型的Jimple和添加的中的局部变量的类型被推断出来。Jimple代码会清除多余的代码，如未使用的变量或赋值。转换到Jimple的一个重要步骤是对表达式进行线性化(和命名)，这样语句最多只引用3个局部变量或常量。从而为执行优化提供更常规和非常方便的表示。在Jimple中，一个分析只需要处理Jimple表示中的15条语句，而Java字节码中可能有200多条指令。
