2020 上海赛

lgtwo

程序分析

libc-2.27

除了没开PIE，其它保护都开了。

菜单题

add

可以选择size，大小在0到0x1000之间，按顺序申请。申请时会让我们输入0x20个内容，内容不存在堆块上，而是存在全局变量上。有一个全局指针数组存堆块索引，一个全局数组存size大小。

delete

输入idx，free掉相应内存。size数组仍然保留数字。

show

假的，这题没有直接输出点，需要爆破stdout。

edit

输入idx，可以往堆里写东西，这里有off-by-one漏洞

攻击思路

有off-by-one，堆块大小任意选，且没开PIE，可以非常容易地构造重叠堆块，实现任意写。但问题在于没有输出的函数，不知道往哪里写东西。

这时通过溢出将堆伪造到IO结构体上，改_IO_write_base，令其泄露出libc基址，具体过程为：

  1. 构造出两个重叠的堆块，申请堆块A,B,C。溢出A，改B的size为size(B)+size(C)。delete(B)，让它进入unsorted bin，两次add，又得到一个指向C的指针
  2. 把在tcache中的堆块放进unsorted bin，间接修改tcache的fd指针。（最终要用tcache分配，所以要利用unsorted bin来改tcahce的 fd）。delete C一次，溢出B覆盖C为unsorted bin的大小，再delete C，令其进入unsorted bin，实现目的。
  3. add 一个小堆块（大小不等于A,B,C)，从unsorted bin中取出，edit 传入**p16(0x6760)**，改低两位字节，这里有4位的爆破。
  4. 将stdout的write_base改为0后，会返回一大串数，可以通过它泄露出libc_base的地址。
  5. 利用之前tcache中的堆块和unsorted bin中重叠的堆块，double free，改fd指针，伪造堆块到freehook处，将其改为system。
  6. free掉一个堆块。
